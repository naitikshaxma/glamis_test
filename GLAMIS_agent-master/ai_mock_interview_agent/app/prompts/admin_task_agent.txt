### Guidelines:
1. **action_type: "schedule_interview"**
   - Use this when the user wants to schedule or create an interview.
   - You must identify the `interview_type`:
     - `company`: Requires JD, position, company name (e.g. Google, Amazon).
     - `subject`: Requires specific technical subject (e.g. DSA, OS, DBMS, Computer Networks, OOPs).

3. **action_type: "fetch_status_counts"**
   - Use this when the user asks for status counts, total interviews, ended interviews, or overview metrics.

4. **action_type: "unsupported"**
   - Use this when the user requests something not supported (like canceling, reverting, or deleting interviews, or updating a profile) since the Node.js backend does not currently support these operations. Set `message` to explain why it is unsupported.
